##  Objectif du tableau de bord

Dans le cadre de ce projet, une interface web a été développée afin de faciliter l’interaction entre l’utilisateur et le système de recommandation.

L’objectif principal de ce tableau de bord est de :

- Permettre à l’utilisateur de tester facilement le modèle  
- Offrir une visualisation claire et simple des recommandations  
- Rendre le système accessible sans nécessiter de connaissances techniques  

Ainsi, cette interface constitue une couche de présentation qui rend le modèle exploitable dans un contexte réel.

---

##  Description de l’interface

L’interface du tableau de bord a été conçue de manière simple et intuitive.

Elle comprend les éléments suivants :

- **Champ User ID** : permet de saisir l’identifiant de l’utilisateur  
- **Champ Product ID** : permet de saisir l’identifiant du produit  
- **Bouton de prédiction** : lance le processus de recommandation  
- **Zone d’affichage** : affiche le résultat de la prédiction  

Cette structure permet une utilisation rapide et efficace du système.

---

##  Fonctionnement général

Le fonctionnement du tableau de bord repose sur une interaction entre l’utilisateur, l’interface et le modèle de recommandation.

Les étapes sont les suivantes :

1. L’utilisateur saisit un **User ID** et un **Product ID**  
2. Il clique sur le bouton de prédiction  
3. Une requête est envoyée au modèle (via une API ou directement en backend)  
4. Le modèle traite les données et génère une prédiction  
5. Le résultat est renvoyé à l’interface  
6. La prédiction est affichée à l’écran  

Ce processus permet une interaction fluide entre l’utilisateur et le système.

---

##  Résultat

À travers ce tableau de bord, l’utilisateur peut :

- Visualiser les prédictions du modèle  
- Tester différents utilisateurs et produits  
- Comprendre le comportement du système de recommandation  

Le dashboard joue donc un rôle essentiel en :

- Simplifiant l’accès au modèle  
- Améliorant l’expérience utilisateur  
- Facilitant la démonstration du projet
